This is a query builder web tool that provides assistance in constructing advanced search queries. It follows a general pattern where the end-user wants to search for some entity that he/she knows nothing about, by providing information about other entities for which he/she knows something.

In the following lines is described a scenario where the following query is constructed through the UI.

“I am looking for the publications of a person named ”Anna”, in any available repository, whose last name I am not sure. The publications should contain the word “VRE” in the title and were published during the period 1/6/2016 and 6/30/2017. Furthermore that “Anna” person is member of an Organization located near Livorno in Italy, the name of which I am not sure”.

Steps:

1.      Select VREs.

 

2.      This is the first view, where we can define what we are looking for through the “Select Target Entity” drop down list.

3.      As soon as a “target” entity is selected, more options are enabled. One of them is that now we are allowed to enter some search keyword to be taken into account during the final search. Thus we can enter the word “vre” in the “Containing Keyword” field. Autocomplete assistance is provided when typing keywords. To enter a keyword that is not recommended through the autocomplete service, we just press enter.

4.      The general concept is that we are seeking for target entity (something that we don’t know yet), considering that it has some relation with some other entity (we call that “related entity”). “Related entity” and the “relation” between target end related entities, can be set through the right panel which is divided into two smaller sub-panels for each one respectively. Right panel becomes enabled after selecting target entity. We can now select the relation “has author”, the time period of interest and the related entity “Person”.

5.      The system allows specifying actual instances for the related entity (“Person” in our case).  To do so, we type the word “Anna” and select the respective recommended search-term from, autocomplete service. Then we click on the “” icon and the system will search for relevant instances to display.

6.      Then we click on the “” icon and the system will search for relevant instances to display. The results of this short search appear on a separate dialog on the screen and are split in pages. We can now select individual person instances by clicking them one by one.

7.      In our case we don’t know the last name of person “Anna”, thus we can check the “Select All” checkbox to search in all of them (Just to remind you that our purpose is to construct the query to be run). The dialog can be closed by pressing the “Select” button.

8.       After closing the dialog we can see the notification “All instances selected”.

9.      To continue with our scenario, we should now use the UI to capture the sentence:

“Anna” person is member of an Organization located near pizza in Italy, the name of which I am not sure”.

To do so we have to add a filter on Person “Anna”. That is done by pressing on the “” icon.

In general, to add filter on the related entity we use the “” icon and to add a new filter on the target entity, we use the “” button and then we either select “OR” or “AND” to define whether the new filter will be applied uniting or intersecting the results of the target entity).

10.   Now the view has been enriched with extra UI for the “filter – related” entity that has been just added. We can now select relation (for filter) “is member of” and related entity (for filter) “Organization Unit”.

11.   In case that the related entity “under inspection” has geospatial nature, a map can be used for geographically restricting the results while seeking for instances. The map can be opened by pressing on the “” icon (only available if there is geospatial nature on that entity).

12.   The region of interesting can be drown by holding the “Ctrl” key and dragging on the map, forming a rectangle. In our case we draw a rectangle around “Livorno”.

13.   Dropped pins denote instances of the related entity under investigation. In the picture below, the data used are synthetic and thus the geographical coordinates used for the pins are fictional.

14.   Hovering over a pin will reveal information about what it represents by popping a caption above it. The caption can be hidden by hovering on another pin or by clicking anywhere on the map but a pin.

15.   Selecting instances can be achieved by clicking on the pins. Clicking on an already clicked pin will de-select the instance. We can select one or more pins to be used as instances in our query. However, in our case we know that we are looking very close to Livorno, and there is only one pin there. So we select it.

16.   As soon as the desired instances have been selected, the map can be closed by clicking anywhere outside the dialog. The selected instances will appear on the right side panel in a new section. That same section can hold instances selected from the key-word search described earlier (if we have applied any).

17.   That’s it. Our query described on the very top of this document is now captured by the UI. The final step is to press the “Search” button located at the bottom right of the view. However that functionality is not yet available.
